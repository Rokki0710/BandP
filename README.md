# B&P prod — Сервис бронирования для съемок и техники

B&P prod — веб-приложение для профессионального медиасообщества. Платформа позволяет арендовать студии, оборудование и бронировать профессиональных фотографов и видеографов. Проект состоит из backend-части на Django REST Framework, frontend-части на HTML/CSS/JavaScript и Telegram-бота для уведомлений.

Проект выполняется в рамках учебной дисциплины, форма обучения — очная, команда 3 студента.

---

## Архитектура проекта

Проект выполнен в виде монолитного Django-приложения с разделением на функциональные модули:

- **users** — регистрация, авторизация, профили пользователей
- **catalog** — каталог студий, оборудования и специалистов
- **bookings** — бронирования, проверка доступности, статусы
- **notifications** — Telegram-бот для отправки уведомлений

---

## База данных

### Модель User (приложение users)
- id (автоинкремент)
- username (уникальное)
- email
- password (хэш)
- first_name
- last_name
- phone
- is_client (булево, клиент/исполнитель)
- date_joined

### Модель Studio (приложение catalog)
- id
- name
- address
- price_per_hour
- description
- image (ссылка на фото)
- owner (связь с User)
- created_at

### Модель Equipment (приложение catalog)
- id
- name
- type (фото/видео/свет/звук)
- price_per_hour
- description
- image
- owner (связь с User)
- created_at

### Модель Specialist (приложение catalog)
- id
- name
- specialization (фотограф/видеограф)
- price_per_hour
- description
- portfolio (ссылка)
- owner (связь с User)
- created_at

### Модель Booking (приложение bookings)
- id
- user (связь с User, кто бронирует)
- studio (связь с Studio, опционально)
- equipment (связь с Equipment, опционально)
- specialist (связь с Specialist, опционально)
- booking_date
- start_time
- end_time
- total_price
- status (pending/confirmed/cancelled/completed)
- created_at

# Лабораторные работы

## Лабораторная работа №1. Базовый скелет проекта и API

**Срок:** 2-3 недели  
**Цель:** Создание базовой структуры проекта, настройка окружения, разработка API для пользователей и каталога, базовая верстка фронтенда с формой авторизации, Telegram-бот с командой /start.

---

### Тиджани — Backend / DevOps

**Задачи:**
1. Создание Django-проекта `bpprod` и приложений `users`, `catalog`, `bookings`, `notifications`.
2. Настройка базы данных SQLite.
3. Создание моделей:
   - `User` — расширение AbstractUser с добавлением полей `phone`, `is_client`
   - `Studio` — название, адрес, цена, описание, изображение, владелец
   - `Equipment` — название, тип, цена, описание, изображение, владелец
   - `Specialist` — имя, специализация, цена, описание, портфолио, владелец
4. Создание миграций и применение их к базе данных.
5. Установка Django REST Framework.
6. Создание ViewSet и сериализаторов:
   - `UserViewSet` — регистрация, список пользователей (только для админа)
   - `StudioViewSet`, `EquipmentViewSet`, `SpecialistViewSet` — только GET-запросы для всех, полный CRUD для владельцев
7. Настройка маршрутов API.
8. Настройка pre-commit hooks (black, flake8).

**Результат:**
- Работающий Django-проект `bpprod`
- Созданные модели и миграции
- API для чтения каталога и регистрации пользователей
- Настроенные git hooks

---

### Почепин — Telegram-бот

**Задачи:**
1. Регистрация бота в BotFather, получение токена.
2. Установка библиотеки python-telegram-bot.
3. Создание файла `bot.py` в приложении `notifications`.
4. Написание бота с командой `/start`.
5. Функция `send_message(chat_id, text)` для отправки сообщений.
6. Логирование событий бота в консоль.

**Результат:**
- Работающий Telegram-бот
- Команда `/start` с приветствием
- Функция отправки сообщений

### Экперов — Frontend

**Задачи:**
1. Создание папки `frontend` в корне проекта.
2. Создание HTML-страниц:

5. Отображение списка студий на странице каталога.

**Результат:**
- Статические HTML-страницы с формами авторизации и регистрации
- Подключение к API для регистрации, входа и чтения каталога
- Отображение каталога

## Лабораторная работа №2. Основной функционал и интеграция

**Срок:** 2-3 недели  
**Цель:** Реализация JWT-аутентификации, создание бронирований, отправка уведомлений в Telegram, интерактивный фронтенд с сохранением авторизации.

---

### Тиджани — Backend / DevOps

**Задачи:**
1. Установка djangorestframework-simplejwt.
2. Создание эндпоинтов для получения и обновления JWT-токена:
   - `/api/token/` — получение токена
   - `/api/token/refresh/` — обновление токена
3. Добавление разрешений (permissions):
   - Только авторизованные пользователи могут создавать бронирования
   - Владельцы могут редактировать свои объекты в каталоге
4. Реализация полного CRUD для каталога.
5. Создание эндпоинта для бронирования:
   - POST `/api/bookings/`
   - Проверка доступности времени (нет пересечений с другими бронями)
   - Автоматический расчет стоимости
   - Создание записи в базе данных
6. Создание эндпоинта для получения бронирований текущего пользователя:
   - GET `/api/bookings/my/`
7. Настройка Docker:
   - Dockerfile для Django-приложения
   - docker-compose.yml с сервисами django и db (PostgreSQL)
8. Написание 3 unit-тестов (создание пользователя, создание брони, проверка доступности).

**Результат:**
- JWT-аутентификация
- Полноценное API для бронирований
- Контейнеризация приложения
- Базовые тесты

---

### Почепин — Telegram-бот

**Задачи:**
1. Интеграция бота с Django-приложением через сигналы.
2. Создание сигнала в приложении `bookings`:
   - При создании объекта Booking вызывается функция отправки уведомления
3. Формирование сообщения:
   ```
   🔔 Новая бронь!
   
   Услуга: [название студии/оборудования/специалиста]
   Дата: [дата]
   Время: [начало] - [конец]
   Клиент: [имя]
   Телефон: [телефон]
   Стоимость: [цена] ₽
   Статус: ожидает подтверждения
   ```
4. Отправка сообщения в фиксированный Telegram-чат (ID чата берется из переменных окружения).
5. Обработка ошибок отправки (бот не падает при ошибке, запись в лог).

**Результат:**
- Автоматическая отправка уведомлений при создании брони
- Информативные сообщения со всеми деталями
- Обработка ошибок
