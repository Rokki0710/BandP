# B&P prod — Сервис бронирования для съемок и техники

B&P prod — веб-приложение для профессионального медиасообщества. Платформа позволяет арендовать студии, оборудование и бронировать профессиональных фотографов и видеографов. Проект состоит из backend-части на Django REST Framework, frontend-части на HTML/CSS/JavaScript и Telegram-бота для уведомлений.

Проект выполняется в рамках учебной дисциплины, форма обучения — очная, команда 3 студента.

---

## Архитектура проекта

Проект выполнен в виде монолитного Django-приложения с разделением на функциональные модули:

- **users** — регистрация, авторизация, профили пользователей
- **catalog** — каталог студий, оборудования и специалистов
- **bookings** — бронирования, проверка доступности, статусы
- **notifications** — Telegram-бот для отправки уведомлений

## База данных

### Модель User (приложение users)
- id (автоинкремент)
- username (уникальное)
- email
- password (хэш)
- first_name
- last_name
- phone
- is_client (булево, клиент/исполнитель)
- date_joined

### Модель Studio (приложение catalog)
- id
- name
- address
- price_per_hour
- description
- image (ссылка на фото)
- owner (связь с User)
- created_at

### Модель Equipment (приложение catalog)
- id
- name
- type (фото/видео/свет/звук)
- price_per_hour
- description
- image
- owner (связь с User)
- created_at

### Модель Specialist (приложение catalog)
- id
- name
- specialization (фотограф/видеограф)
- price_per_hour
- description
- portfolio (ссылка)
- owner (связь с User)
- created_at

### Модель Booking (приложение bookings)
- id
- user (связь с User, кто бронирует)
- studio (связь с Studio, опционально)
- equipment (связь с Equipment, опционально)
- specialist (связь с Specialist, опционально)
- booking_date
- start_time
- end_time
- total_price
- status (pending/confirmed/cancelled/completed)
- created_at

# B&P prod — Инструкция по запуску

## Требования

- Python 3.10+
- Docker и Docker Compose (или SQLite для локального запуска)
- Node.js 16+ (для фронтенда)

---

## 🐳 Запуск с Docker (рекомендуется)

### 1. Клонировать репозиторий
```bash
git clone <url-репозитория>
cd bpprod
```

### 2. Создать файл окружения
```bash
cp .env.example .env
```

### 3. Запустить контейнеры
```bash
docker-compose up -d
```

### 4. Применить миграции
```bash
docker-compose exec web python manage.py migrate
```

### 5. Создать суперпользователя (админа)
```bash
docker-compose exec web python manage.py createsuperuser
```

### 6. Запустить Telegram бота
```bash
docker-compose exec web python manage.py runbot
```

### 7. Открыть в браузере
- **API документация:** http://localhost:8000/api/docs/
- **Админ-панель:** http://localhost:8000/admin/
- **Фронтенд:** http://localhost:5173/

### 8. Остановить контейнеры
```bash
docker-compose down
```

## 💻 Локальный запуск (без Docker)

### Backend

1. **Создать виртуальное окружение**
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows
```

2. **Установить зависимости**
```bash
pip install -r requirements.txt
```

3. **Настроить SQLite (отредактировать settings.py)**
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
```

4. **Применить миграции**
```bash
python manage.py migrate
```

5. **Создать суперпользователя**
```bash
python manage.py createsuperuser
```

6. **Запустить сервер**
```bash
python manage.py runserver
```

### Frontend

7. **Установить и запустить фронтенд**
```bash
cd frontend
npm install
npm run dev
```

### Telegram Bot

8. **Запустить бота**
```bash
python manage.py runbot
```

##  Устранение проблем

### CORS ошибка при регистрации
Добавьте в `settings.py`:
```python
CORS_ALLOWED_ORIGINS = [
    'http://localhost:5173',
    'http://127.0.0.1:5173',
]
```

### Ошибка "password2 is required"
Фронтенд должен отправлять оба поля:
```javascript
{
    "password": "password",
    "password2": "password"  // обязательно!
}
```

### Ошибка DisallowedHost

##  API Эндпоинты

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/token/` | Вход (получение токена) |
| POST | `/api/register/` | Регистрация |
| GET | `/api/studios/` | Список студий |
| GET | `/api/equipment/` | Список оборудования |
| GET | `/api/specialists/` | Список специалистов |
| POST | `/api/bookings/` | Создать бронь |
| GET | `/api/bookings/my/` | Мои брони |
| GET | `/api/docs/` | Swagger документация |
